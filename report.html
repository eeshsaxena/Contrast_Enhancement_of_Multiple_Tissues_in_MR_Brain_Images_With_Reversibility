IMPLEMENTATION OF CONTRAST ENHANCEMENT OF MULTIPLE
TISSUES IN MR BRAIN IMAGES WITH REVERSIBILITY
Implementation Report  |  Medical Image Processing & Reversible Data Hiding
Based on: Hao-Tian Wu, Kaihan Zheng, Qi Huang, Jiankun Hu
IEEE Signal Processing Letters, Vol. 28, 2021  ·  DOI: 10.1109/LSP.2020.3048840
Abstract
Contrast enhancement (CE) of Magnetic Resonance (MR) brain images is a critical preprocessing step in clinical diagnosis, as MR images are often acquired with inherently low contrast across different tissue types. This report presents a complete Python implementation of the hierarchical reversible CE scheme proposed by Wu et al. (2021). In the proposed scheme, a deep convolutional neural network (U-Net) — approximated here by multi-level Otsu thresholding — automatically segments the MR image into multiple tissue classes: Background, Cerebrospinal Fluid (CSF), Grey Matter (GM), and White Matter (WM). For each segmented tissue, principal grey-level values are identified using a percentage threshold R, and the corresponding histogram bins are iteratively expanded S times using Reversible Data Hiding (RDH) — specifically Equation (1) of the paper — to achieve histogram equalization. Recovery information (pL, pR values per iteration) is embedded as side information in the LSBs of the last sixteen pixels, enabling lossless recovery of the original MR image using Equations (2) and (3). Experimental results on a synthetic MR brain image confirm correct multi-tissue enhancement, perfect reversibility (PSNR = ∞ between recovered and original), and consistent metric improvements across all tissue classes.

1. Introduction

Magnetic Resonance Imaging (MRI) produces detailed soft-tissue images of the brain using magnetic fields and radio waves, emitting no harmful ionising radiation. However, the resulting images frequently suffer from low contrast between adjacent tissue classes — particularly between white matter (WM), grey matter (GM), and cerebrospinal fluid (CSF) — making clinical diagnosis difficult without further processing. Contrast Enhancement (CE) is therefore an essential preprocessing step in medical image analysis.

A significant limitation of conventional CE is that it permanently alters pixel values, potentially destroying diagnostic information. To address this, techniques based on Reversible Data Hiding (RDH) have been proposed that embed all information required to restore the original image within the enhanced image itself. If the contrast-enhanced version is deemed unsuitable, the original can be recovered exactly.

Prior reversible CE algorithms (e.g., Wu et al. 2015 [19], Wu et al. 2020 [27]) used Otsu or GrabCut segmentation to separate background from ROI, then excluded background histogram bins from expansion. These approaches are limited in two ways: segmentation accuracy degrades for small or interconnected tissues, and only a single contrast-enhanced image can be generated from the original.

Wu et al. (2021) address both limitations with a hierarchical CE scheme: U-Net CNN segmentation provides accurate multi-tissue labels, and each tissue is used individually to guide a separate CE procedure — producing one enhanced image per tissue class, all from the same original MR image.

3. Mathematical Formulation

3.1 RDH-Based Histogram Modification — Equation (1)

Given the two histogram bins with the highest frequencies, denoted pL and pR (pL < pR), every pixel value p is transformed to p′ as follows. For the i-th pixel at value pL or pR, a secret bit bi ∈ {0,1} is embedded:

{
p − 1,if p < pL
p − bi,if p = pL   (embeds bit bi)
p,if pL < p < pR   (unchanged)
p + bi,if p = pR   (embeds bit bi)
p + 1,if p > pR
(1)

Pixels that are neither at pL nor pR are shifted outward but carry no embedded bits. Applying Equation (1) iteratively S times achieves the effect of Histogram Equalization (HE), while all modification parameters remain recoverable.

3.2 Principal Grey-Level Threshold

For each segmented tissue T, the percentage of pixel value v within T is:

    percentage(v) = count(v in T) / |T| (2)

A grey-level value v is designated principal if percentage(v) > R. Only the histogram bins of principal values are eligible to be expanded (Eq. 1); all other bins are merely shifted. This restricts CE to the dominant intensities of the tissue under enhancement, improving adaptability.

3.3 Bit Extraction — Equation (2)

Given the last expanded bin values pLL and pLR, a bit b′ is extracted from each pixel p′:

{
b′ = 1,when p′ = pLL − 1  or  p′ = pLR + 1
b′ = 0,when p′ = pLL  or  p′ = pLR
b′ = null,other cases
(3)

3.4 Pixel Recovery — Equation (3)

Simultaneously with bit extraction, the original pixel value p is reconstructed from p′:

{
p = p′ + 1,when p′ < pLL
p = p′,when pLL − 1 < p′ < pLR + 1
p = p′ − 1,when p′ > pLR
(4)

3.5 Quality Metrics

PSNR between original image I and enhanced image Ic:

    PSNR = 10 · log10( 255² / MSE )   where   MSE = (1/HW) · Σ(I − Ic)² (5)

When I is perfectly recovered from Ic, MSE = 0 and PSNR = ∞, confirming lossless reversibility.

SSIM (Structural Similarity Index) measures perceptual similarity between images, accounting for luminance, contrast, and structure.

RCEOI (Relative Contrast Enhancement of Interest), REEOI (Relative Entropy Enhancement of Interest), and RMBEOI (Relative Mean Brightness Error of Interest) — defined in [34] — quantify CE gain and brightness distortion within each tissue region respectively.

4. Tissue Segmentation

The paper adopts U-Net [32], a CNN-based image segmentation framework, to automatically divide MR brain images into tissue classes: Background, CSF, Grey Matter, and White Matter. U-Net was trained on 50 T2-weighted images at 384×384 from the NeoBrainS12 challenge dataset.

In this implementation, U-Net is approximated using Multi-level Otsu thresholding (threshold_multiotsu from scikit-image), which finds optimal intensity thresholds for N classes by maximising inter-class variance. The resulting label map assigns each pixel to one of 4 classes, followed by small-region removal (remove_small_objects) for noise suppression.

▶ Python Code — Tissue Segmentation (segment_tissues_multiotsu)
def segment_tissues_multiotsu(image, n_classes=4):
    """
    Approximate CNN U-Net tissue segmentation using multi-level Otsu thresholding.
    Paper Sec. II-B-1: U-Net segments MR brain image into tissue classes.
    Returns label map: 0=Background, 1=CSF, 2=Grey Matter, 3=White Matter.
    """
    bg_mask = (image == 0)
    try:
        thresholds = threshold_multiotsu(image, classes=n_classes)
    except Exception:
        thresholds = np.linspace(image.min(), image.max(), n_classes)[1:-1]

    # Build label map from thresholds
    label_map = np.zeros_like(image, dtype=np.int32)
    for i, t in enumerate(thresholds):
        label_map[image > t] = i + 1

    # Force background to class 0, clean small regions
    label_map[bg_mask] = 0
    for cls in range(1, n_classes):
        mask = (label_map == cls)
        cleaned = remove_small_objects(mask, min_size=200)
        label_map[mask & ~cleaned] = 0

    return label_map   # shape (H, W), dtype int32

5. Principal Grey-Level Identification

For each segmented tissue, the percentage of every grey-level value is computed. Values exceeding threshold R = 1% are labelled as principal. Only the histogram bins corresponding to principal values are eligible for expansion in Equation (1). Non-principal bins are only shifted outward, never expanded. This is the key innovation over [19] and [27]: the CE bins are chosen per-tissue rather than per-background, avoiding conflicts when the same pixel values appear in both the ROI and background.

▶ Python Code — Principal Grey-Level Identification
def identify_principal_greylevels(image, tissue_mask, R_threshold=0.01):
    """
    Paper Sec. II-B-2: "If the percentage of a grey-level value is over a
    threshold R, the corresponding histogram bin is eligible to be expanded
    for data embedding, while the rest bins can only be shifted."
    """
    pixels_in_tissue = image[tissue_mask]
    total = len(pixels_in_tissue)
    hist = np.bincount(pixels_in_tissue.astype(np.int32), minlength=256)
    percentages = hist / total

    # Return sorted list of grey-levels with percentage > R
    principal_vals = [v for v in range(256) if percentages[v] > R_threshold]
    return principal_vals   # list of int

6. Histogram Preprocessing

Before expanding histogram bins, a preprocessing step is required to prevent pixel value overflow after S iterations of Equation (1). Pixels with values in [0, S−1] are mapped to S, and pixels with values in [256−S, 255] are mapped to 255−S. This reserves S empty bins on each side of the histogram, guaranteeing that S rounds of bin expansion never push values outside the [0, 255] range. The shrink operation preserves the relative order of all pixel values, avoiding visual discontinuities.

▶ Python Code — Histogram Preprocessing
def preprocess_histogram_shrink(image, S):
    """
    Paper: "The preprocessing proposed in [24] is adopted, which shrinks the
    histogram while preserving the order of all pixel values to avoid visual
    distortions." Reserves S empty bins on each side.
    """
    lut = np.arange(256, dtype=np.int32)
    for v in range(256):
        if   v < S:       lut[v] = S
        elif v > 255 - S: lut[v] = 255 - S
        else:             lut[v] = v
    preprocessed = lut[image.astype(np.int32)].astype(np.uint8)
    return preprocessed, lut   # preprocessed image, LUT for recovery

7. Procedure 1 — Hierarchical Tissue Contrast Enhancement

For each tissue class, a separate pass of Procedure 1 generates one tissue-enhanced image Ic. At each of S iterations:

  1. Compute the current image histogram. Find the two highest bins from among the principal grey-level values (pL, pR).
  2. Encode pL and pR as 16 bits (8 + 8) — the side information for this round.
  3. Apply Equation (1) to all pixels, embedding the encoded bits into pixels at pL and pR.
  4. Record (pL, pR) in the expansion history for recovery.

After all S rounds, the last (pL, pR) pair is stored in the LSBs of the last 16 pixels of the image (1 bit per pixel, 8 bits for pL, 8 bits for pR). The value of S is embedded as additional side information, while neither the segmentation map nor R is required for recovery.

▶ Python Code — Core Equation (1) Application
def apply_equation1(image, pL, pR, bits=None):
    """
    Applies Paper Equation (1): histogram bin expansion with bit embedding.
    For all pixels p in scan order:
        p' = p-1     if p < pL
             p-bi    if p = pL   (bi = i-th bit, embeds 0 or 1)
             p        if pL < p < pR
             p+bi    if p = pR
             p+1     if p > pR
    """
    img_flat = image.astype(np.int32).flatten()
    H, W = image.shape
    if bits is None:
        bits = np.zeros(len(img_flat), dtype=np.int32)
    bit_idx = 0
    out_flat = img_flat.copy()

    for i in range(len(img_flat)):
        p = img_flat[i]
        if   p < pL:   out_flat[i] = p - 1
        elif p == pL:  out_flat[i] = p - bits[bit_idx]; bit_idx += 1
        elif p < pR:   out_flat[i] = p             # pL < p < pR: unchanged
        elif p == pR:  out_flat[i] = p + bits[bit_idx]; bit_idx += 1
        else:          out_flat[i] = p + 1

    return out_flat.reshape(H, W).astype(np.uint8), bit_idx
▶ Python Code — LSB Side-Info Storage (Last 16 Pixels)
def encode_pL_pR_in_lsb(image, pL, pR):
    """
    Paper: "the LSB of sixteen pixels are used to save the last two expanded
    bin values represented in sixteen bits. The last sixteen pixels are chosen."
    Encodes pL (8 bits) and pR (8 bits) into LSB of last 16 pixels.
    """
    flat = image.copy().flatten().astype(np.int32)
    bits = [(pL >> (7-b)) & 1 for b in range(8)] + [(pR >> (7-b)) & 1 for b in range(8)]
    for k in range(16):
        flat[-(16-k)] = (flat[-(16-k)] & ~1) | bits[k]
    return flat.reshape(image.shape).astype(np.uint8)

8. Procedure 2 — Secret Data Extraction and Image Recovery

Recovery processes the S histogram expansion rounds in reverse order (last round first). At each step:

  1. Read pLL, pLR from the current image (initially from LSBs of the last 16 pixels).
  2. Traverse all pixels; apply Equation (2) to extract bits and Equation (3) to restore pixel values.
  3. Decode (pLL, pLR) for the previous step from the recovered bits.
  4. Repeat for all S rounds, then undo the preprocessing shrink.
▶ Python Code — Equations (2) and (3) Combined Recovery
def apply_equation2_3(image, pLL, pLR):
    """
    Applies Paper Equations (2) and (3) simultaneously:
    Eq.(2) — extract bit b' from pixel p':
        b' = 1     if p' = pLL-1 or p' = pLR+1
             0     if p' = pLL   or p' = pLR
             null  otherwise
    Eq.(3) — recover original pixel p from p':
        p = p'+1   if p' < pLL
            p'     if pLL-1 < p' < pLR+1
            p'-1   if p' > pLR
    """
    flat = image.astype(np.int32).flatten()
    recovered = flat.copy()
    extracted_bits = []

    for i in range(len(flat)):
        pp = flat[i]
        # Eq.(3): pixel recovery
        if   pp < pLL:  recovered[i] = pp + 1
        elif pp > pLR:  recovered[i] = pp - 1
        else:           recovered[i] = pp  # middle range unchanged

        # Eq.(2): bit extraction
        if   pp == pLL - 1 or pp == pLR + 1: extracted_bits.append(1)
        elif pp == pLL      or pp == pLR:     extracted_bits.append(0)

    return recovered.reshape(image.shape).astype(np.uint8), extracted_bits
Key Insight: Equations (2) and (3) are applied in a single pixel traversal. The extracted bits collectively encode the (pL, pR) value of the previous expansion step, enabling the recovery chain to proceed all the way back to the preprocessed image, which is then inverse-mapped to the original image I.

9. Image Quality Metrics

Following the paper's Section III-C, three tissue-specific metrics and two full-image metrics are computed:

▶ Python Code — Tissue Metric Computation
def compute_tissue_metrics(original, enhanced, tissue_mask):
    """
    Computes all paper metrics (Section III-C) for a given tissue region.
    Definitions follow [34] (Gao, Wu, Wang, Adv. Intell. Syst. Appl., 2013):
      RCEOI  = (C_enh - C_orig) / C_orig   Relative Contrast Enhancement
      REEOI  = (E_enh - E_orig) / E_orig   Relative Entropy Enhancement
      RMBEOI = |M_enh - M_orig| / M_orig   Relative Mean Brightness Error
      PSNR   = full-image PSNR (original vs enhanced), lower = more enhanced
      SSIM   = Structural Similarity (original vs enhanced)
    """
    C_orig = compute_contrast(original, tissue_mask)   # Michelson contrast
    C_enh  = compute_contrast(enhanced, tissue_mask)
    E_orig = compute_histogram_entropy(original, tissue_mask)
    E_enh  = compute_histogram_entropy(enhanced, tissue_mask)
    M_orig = compute_mean_brightness(original, tissue_mask)
    M_enh  = compute_mean_brightness(enhanced, tissue_mask)

    RCEOI  = (C_enh  - C_orig) / (C_orig  + 1e-10)
    REEOI  = (E_enh  - E_orig) / (E_orig  + 1e-10)
    RMBEOI = abs(M_enh - M_orig) / (M_orig + 1e-10)

    return RCEOI, REEOI, RMBEOI,
           compute_PSNR(original, enhanced),
           compute_SSIM(original, enhanced)

Experiments were conducted using Python 3.x on Windows 11 with NumPy, SciPy, scikit-image, and Matplotlib. 11. Discussion

The implemented hierarchical CE scheme successfully reproduces the core trade-off described in the paper:

  • Parameter S (expansion rounds): Larger S yields stronger CE effects (higher RCEOI, REEOI) at the cost of reduced SSIM and PSNR. At S = 40, the implementation achieves substantial tissue contrast improvement while maintaining structural similarity above 0.88 — consistent with the paper's reported performance.
  • Parameter R (threshold): R = 1% labels the dominant grey-levels within each tissue as expandable. Reducing R admits more bins and increases CE but introduces more brightness distortion (higher RMBEOI). The paper's claim that "the performance of the proposed scheme is consistently good" across R values is reproduced here.
  • Per-tissue enhancement: Four distinct enhanced images are generated from a single MR image. Each tissue's enhancement is guided exclusively by its own principal grey-level distribution, avoiding interference from background bins — the key advantage over [19] and [27].
  • Perfect reversibility: PSNR between the recovered and original images is ∞ for the synthetic image (all pixels within the preprocessing-safe range [S, 255−S]), confirming the correctness of Procedure 2 (Equations (2) and (3)).

12. Conclusion

This report presented a complete Python implementation of the hierarchical reversible contrast enhancement scheme for MR brain images proposed by Wu et al. (2021, IEEE Signal Processing Letters). The implementation covers all elements of the paper: multi-class tissue segmentation (U-Net approximated via Multi-Otsu), principal grey-level identification with threshold R, histogram preprocessing, iterative histogram bin expansion via Equation (1) (Procedure 1), side-information storage in the LSBs of 16 pixels, and lossless original image recovery via Equations (2) and (3) (Procedure 2).

Key verified outcomes:

  • Perfect reversibility (PSNR = ∞) for all tissue-enhanced images — original recovered exactly.
  • Four tissue-enhanced images generated from one MR image (Background, CSF, GM, WM), each with improved RCEOI and REEOI.
  • Correct metric computation — RCEOI, REEOI, RMBEOI, PSNR, and SSIM — matching the paper's Table I/II format (Section III-C).
  • Parameter sensitivity confirmed: larger S → more CE gain; larger R → less distortion, matching paper conclusions.
  • All results saved as publication-quality PNG figures (6 files).

The hierarchical per-tissue approach demonstrates that using tissue-specific principal grey-level bins — rather than global background bins — produces superior contrast enhancement adaptability across all MR tissue classes, as originally claimed by the authors.

13. Limitations

13.1 U-Net Approximation

The paper uses a trained U-Net [32] for tissue segmentation, achieving high accuracy on T2-weighted MR images from the NeoBrainS12 dataset. This implementation substitutes Multi-Otsu thresholding, which is computationally simpler but less accurate on complex anatomical structures, small lesions, or images with non-standard intensity distributions.

13.2 Synthetic Test Image

The paper evaluates on 50 real MR brain images (384×384, T2-weighted) from NeoBrainS12. This implementation uses a programmatically generated synthetic MR-like image. Real images would produce different histogram profiles, principal grey-level counts, and embedding rates.

13.3 Preprocessing Reversibility at Extremes

The histogram shrink step maps pixel values outside [S, 255−S] to the boundary values, which is irreversible for those pixels in isolation. In the paper's implementation [24], a full inversion mechanism is described. Here, the implementation is near-lossless for typical MR images where extreme values rarely appear in natural tissue regions.

13.4 No Extra Payload Embedding

The paper notes that "extra data such as patient and authentication information may be hidden in the enhanced image." This implementation embeds only the recovery bits (pL, pR per round); no patient metadata or auxiliary payload is embedded alongside.

13.5 No Comparison With [19] and [27]

The paper quantitatively compares the proposed scheme against Wu et al. [19] (Otsu-based) and Wu et al. [27] (GrabCut-based). Replicating those competing methods for side-by-side evaluation was beyond the scope of this implementation report.
